# Налаштування лінтингу для проєкту Network IDS

## Обраний лінтер

### Pylint (основний лінтер)
**Версія:** 3.2.0
**Документація:** https://pylint.pycqa.org/

#### Чому Pylint?
1. **Всебічний аналіз** - перевіряє стиль, помилки, потенційні проблеми
2. **Гнучке налаштування** - можна вимкнути непотрібні правила
3. **Інтеграція з IDE** - добре працює з VS Code
4. **Оцінка коду** - дає числову оцінку якості (0-10)
5. **Розширюваність** - підтримує плагіни

### Black (форматувальник коду)
**Версія:** 24.2.0
**Документація:** https://black.readthedocs.io/

#### Чому Black?
1. **"Не потребує налаштувань"** - автоматичне форматування
2. **Швидкість** - миттєве форматування
3. **Консистентність** - однаковий стиль всього проєкту

## Аспекти якості для проєкту

| Аспект | Пріоритет | Чому важливо |
|--------|-----------|--------------|
| Стиль коду | Високий | Читабельність та підтримуваність |
| Безпека | Високий | IDS має бути захищеним |
| Продуктивність | Середній | Обробка мережевого трафіку |
| Документація | Середній | Зрозумілість коду |

## Базові правила в `.pylintrc`

### Форматування
| Правило | Значення | Пояснення |
|---------|----------|-----------|
| `max-line-length` | 100 | Максимальна довжина рядка |
| `indent-string` | '    ' | Використовуємо 4 пробіли |
| `expected-line-ending-format` | LF | Unix-style кінці рядків |

### Стиль коду
| Правило | Значення | Пояснення |
|---------|----------|-----------|
| `max-complexity` | 12 | Максимальна складність функції |
| `max-args` | 6 | Максимум аргументів функції |
| `max-methods` | 20 | Максимум методів у класі |
| `max-nested-blocks` | 5 | Максимальна глибина вкладеності |

### Імена змінних
| Тип | Патерн | Приклад |
|-----|--------|---------|
| Змінні | `[a-z_][a-z0-9_]{2,30}$` | `network_traffic` |
| Функції | `[a-z_][a-z0-9_]{2,30}$` | `detect_anomalies` |
| Класи | `[A-Z_][a-zA-Z0-9]+$` | `AnomalyDetector` |
| Константи | `[A-Z_][A-Z0-9_]+$` | `MAX_THRESHOLD` |
